## Issue 5 — 异步任务接口（含前端切换）

### 目标
v2 文档 P3。后端从 sync 切换到 job 模型，前端切到提交 + 轮询。前后端必须一起改，所以合并成一个 issue / 一个 PR。

### 主要工作
- **后端**
  - `POST /api/jobs`（创建任务、立即返回 jobId）
  - `GET /api/jobs/:jobId`（查询单任务状态）
  - 单进程 FIFO worker
  - 状态流转 `queued → running → success/failed`
  - 启动时把残留 `running` 标为 `failed`，错误码 `SERVER_RESTARTED`
  - 删除旧 `POST /api/remove-watermark`
- **前端**
  - 提交表单后拿 `jobId`，进入轮询
  - 各状态展示
  - 失败提示失败原因

### ❗️ 待决策
- [ ] **Worker 实现方式：**
  - A) `setInterval` 轮 DB 拉 `queued`（简单、有最长 N 秒启动延迟）
  - B) 内存事件 + DB 持久化（提交时直接唤醒 worker，重启时回退到 A）
- [ ] **前端轮询间隔？** 1s / 2s 固定 还是指数退避
- [ ] **`durationMs` 测什么范围？** 只 IOPaint exec，还是含 Sharp 转换 + 落盘
- [ ] **jobId 类型？** 继续用 uuid v4 还是改 Prisma `cuid()`

### 已定（v2 文档已写）
- 旧接口直接删除，硬切，不做过渡期
