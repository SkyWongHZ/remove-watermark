# 项目迭代路线

本文档用于快速说明项目各阶段目标和对应设计文档，避免把“已实现能力”和“规划中能力”混在一个文件里。

## V1：本地单图同步去水印

状态：已实现

目标：

- 本地单人使用。
- 上传 JPG/PNG/WebP 图片。
- 使用矩形区域标记水印。
- 前端生成 mask。
- 后端同步调用 IOPaint CLI。
- 返回 PNG 结果图。
- 结果图保存在本地 `storage/outputs`。

对应文档：

- [本地图片去水印 MVP 技术方案](./本地图片去水印MVP技术方案.md)
- [API 接口设计](./API接口设计.md)
- [本地安装与运行指南](./本地安装与运行指南.md)

## V2：线上部署 + MySQL + 异步任务

状态：规划中

目标：

- 先将当前单图同步版本部署到阿里云 ECS，验证线上 IOPaint 可用性和处理速度。
- 沉淀线上部署配置、Nginx 配置、进程管理方式和环境变量。
- 接入 MySQL + Prisma，保存任务状态、文件路径、结果 URL、耗时和错误信息。
- 将主流程升级为异步任务接口。
- 前端增加当前任务状态轮询。
- 前端增加最近 20 条简单历史列表。
- 后端预留 OSS 扩展字段和存储接口，但第二版不实际接入 OSS。

对应文档：

- [第二版功能规划与实施建议](./第二版功能规划与实施建议.md)

## V3：候选扩展方向

状态：待评估

候选能力：

- 阿里云 OSS 实际接入。
- 自定义涂抹 mask。
- 批量图片处理。
- IOPaint HTTP 服务模式。
- GPU 推理优化。
- Redis + BullMQ 队列。
- 用户登录和权限。
- 视频去水印。

进入条件：

- V2 的线上单图处理稳定。
- MySQL 任务记录和异步状态流转可靠。
- 明确出现对应的真实使用需求。

## 当前优先级

```text
1. 线上部署当前 V1 同步版本
2. 补齐线上部署文档和配置
3. 接入 MySQL + Prisma
4. 改造为异步任务流程
5. 增加简单历史列表
6. 再评估 OSS、批量、自定义涂抹和视频能力
```
